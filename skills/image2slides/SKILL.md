# Image2Slides

Use this workflow for `/image2slides`. The output is not a flat image-only deck: final text must be editable PowerPoint text layered above non-selectable background images.

## Required Inputs

Before running generation, confirm these fields are present. They are not optional:

- slide base style and color tone
- slide aspect ratio
- slide page count
- slide purpose: `speech` or `showcase`
- presentation scene: academic, enterprise, classroom, life, or another explicit scene
- knowledge base: user-provided text/image/material paths or pasted materials

If any required field is missing, ask only for the missing fields and stop before creating project artifacts.

## Core Contract

1. Create a project directory and write the required intent to disk with:
   ```bash
   python skills/image2slides/scripts/image2slides.py init --project <deck-dir> --spec <spec.json>
   ```
2. Maintain the wiki structure created by `init`. Put factual source material under `wiki/grep/` and generated/story material under `wiki/generate/`.
3. Create `wiki/02_content_boundary.md` before writing prompts. Every slide needs a boundary:
   - `grep_required`: facts, claims, dates, names, quotes, current data, citations, or anything that must come from source/web/search.
   - `generation_allowed`: visual metaphors, transitions, examples, phrasing, non-factual teaching scaffolds, and abstract imagery.
4. Use Codex native `image_gen` as the default GPT-image-2 path. This path does not require a user-supplied API key. The helper CLI writes prompts and file structure; the Codex agent executes native `image_gen`, then copies generated assets from `$CODEX_HOME/generated_images/...` into the deck project.
   - default model path: native GPT-image-2 through `image_gen`
   - completed references: `completed/slide_XX_completed.png` must be generated by GPT-image-2 image_gen as complete slide reference images with visible text
   - provenance: after native image_gen outputs are copied into `completed/`, run `register-completed`; the workflow must reject unregistered PPTX/PDF/render screenshots
   - background references: use image_gen edit on each completed image to produce `background/slide_XX_background.png`; remove only slide text while preserving geometry
   - background provenance: after native image_gen edit outputs are copied into `background/`, run `register-background`; `analyze`, `build-pptx`, `compose-source-locked`, and `qa` must reject unregistered local templates, screenshots, deterministic drawings, or backgrounds not tied to the current completed provenance
   - source-locked patching: after both image_gen batches exist, `compose-source-locked` may paste exact source figures into both images, but it must never create `completed/` from local drawing, PPTX rendering, or screenshots
   - optional API fallback only when explicitly requested: `image2slides imagegen --phase ... --execute`
5. Save full slide images in `completed/` and matching text-free pages in `background/`.
   When a user marks data/results as non-mutating, keep those charts or figures source-locked. They may be placed as exact `source_layers` from the wiki instead of being redrawn as generated facts; the GPT-image-2 pass still owns the surrounding slide composition and matched text-free background.
   When native imagegen already created visual panels, set `draw_frame: false` on matching `source_layers` so source patching does not add duplicate rounded rectangles.
   Source-layer fitting is deterministic and bidirectional: before queueing prompts, run `normalize-source-panels` or rely on `queue` to do it automatically so every source panel bbox is rewritten from the trimmed source image ratio plus constant margin `d` (`fit_margin_px`, default 32 px). Queue prompts reserve those source-aspect blank panels; compose trims blank pixels while preserving content, detects the actual native panel edge when a GPT-image-2 base exists, requires the visible/native panel inset to match the trimmed source ratio, scales the image so each edge is `<=` its parallel inset-panel edge, minimizes remaining `B - A` slack by maximizing scale, and aligns image center to inset-panel center. For generated/native panels, `d` must never be below 32 px even if a plan layer sets a smaller margin, because smaller margins can look geometrically valid while visually pressing the background panel. Edge-connected near-white blank pixels in source figures are pasted transparently by default, so a source chart cannot bring its own white rectangle that covers the generated panel.
   `queue` also mirrors every `source_layers[].panel_bbox` into `layout_boundaries` as `kind: non_editable_image_panel`. Treat that boundary as a hard no-text region for editable PowerPoint text. Text printed inside a source figure, chart, diagram, schematic, or icon is asset-internal text: preserve it as image content and exclude it from editable-text placement diagnostics unless the user explicitly asks to extract it.
   Do not disable panel detection for generated source panels. `detect_panel: false` bypasses the required four-edge check and must fail strict audit unless the slide is explicitly drawing its own deterministic frame. The declared `bbox` or `panel_bbox` is only a search hint and may describe a wrong insertion position; it is never panel truth. Never clear the declared hint region before panel detection, because clearing a wrong hint creates fake borders and makes the wrong paste location look valid. If the detected GPT-image-2 panel ratio does not match the trimmed source ratio, fail the audit and regenerate the source-free background/completed pair; do not locally stretch, crop, redraw, or pretend the wrong-ratio panel is valid. Source insertion must not mutate an already source-locked image pair by default: if provenance already ends with `_with_source_locked_patch`, `compose-source-locked` must stop unless `--force` is explicitly used after review. Never clear or redraw generated panel borders during normal source patching; paste only the trimmed source image by the four-edge `A <= B - 2d` rule.
   Figure-first rule: on any source-led slide, the figure is the protagonist and text is only a supporting reading guide. Keep only source-locked result/data figures as image panels. Do not reserve panels for decorative diagrams, icons, screenshots whose internal text can be extracted, or text-heavy schematics; extract the useful text into editable PowerPoint text or delete it. Use fewer words, give the figure the largest intuitive visual zone, and avoid crowding multiple panels unless each panel is necessary to preserve a result.
6. Run analysis:
   ```bash
   python skills/image2slides/scripts/image2slides.py analyze --project <deck-dir>
   ```
   This compares `completed/` and `background/`, treats their pixel difference as the text mask, identifies dominant background color, surrounding pixel variation, text regions, and blank fill regions.
7. Build the PPTX:
   ```bash
   python skills/image2slides/scripts/image2slides.py build-pptx --project <deck-dir>
   ```
   Each background image is written as the slide background layer, then editable text boxes are placed above it.
   `build-pptx` runs `lint-visible` first and fails if control-plane metadata appears in visible slide text.
8. Verify:
   ```bash
   python skills/image2slides/scripts/image2slides.py qa --project <deck-dir> --strict
   ```
   This re-renders the PPTX when local tools are available, compares each rendered page with `completed/`, and writes pixel/patch similarity reports. Do not accept a report produced from stale rendered screenshots.
   The default strict similarity preset is for editable PowerPoint text over imagegen backgrounds: overall pixel similarity must stay at or above `0.90`, while bad-pixel and patch tolerances allow expected font rasterization differences between GPT-image-2 reference text and native PPT text. Do not relax provenance, background-duplicate, source-layer, or stale-render checks.
   QA also writes `reports/source_layer_audit.md`, `reports/text_alignment_audit.md`, `reports/source_render_audit.md`, and `reports/background_audit.md`. These fixed review items check that source figures stay inside detected/native panels, the visible/native panel inset ratio matches the trimmed source image ratio, source figures/panels do not overlap each other or editable text, rendered source crops match the final source-locked background crop, source transparent blank areas still match the source-free native panel, editable text crops align locally against completed references, figure-first source area dominates text area, figure-first text is concise, duplicate rounded-rectangle frames are not added over native imagegen panels, and background pages are neither byte-identical nor visually near-identical duplicates.
   QA also writes `reports/content_boundary_audit.md` and overlay images under `reports/content_boundary_overlays/`. This audit detects each background's main blank color, defines `blank_zone`, `forbidden_zone`, and `text_fill_zone`, treats plan-derived source panels as non-editable regions, excludes asset-internal source/illustration labels from editable text masks, and reports whether PowerPoint text falls outside the blank fill zone or too close to source panels/illustrations.
   QA must fail strict mode when source panel detection is disabled, when a source image is not inset from all four detected panel edges by margin `d`, or when the pasted image center does not match the inset panel center.
   The audit records `declared_panel_bbox`, `detected_panel_bbox`, `actual_panel_bbox`, `visible_panel_bbox`, `panel_bbox`, `fit_bbox`, `paste_bbox`, source and panel aspect ratios, `margin_px`, `slack_px`, and `center_delta_px` for every source layer. Strict failures include `figure_not_primary`, `text_dominates_figure`, and `figure_slide_text_overload`.
   Use `image2slides qa --project <deck-dir> --boundary-strict` only when you want blank-zone warnings to fail the command. Without `--boundary-strict`, the boundary audit is diagnostic evidence for Codex native LLM review and the final human detail pass.
9. Perform a brief human detail check after strict QA. Open the final PPTX and scan figure-panel padding, line breaks, font scale, obvious text overflow, page-to-page consistency, and whether source data/results remain unchanged. Treat this as a final polish pass, not as a substitute for strict QA. When the deck is accepted, export the reviewed PowerPoint directly to `pptx/image2slides.pdf` as the canonical visual snapshot so later render/compile conversions do not become the only inspection artifact.

## Prompt Rules

The required input fields are control-plane metadata. They shape the deck, but they must not appear as visible slide copy. Do not render phrases such as the chosen tone, aspect ratio, page count, purpose, scene label, knowledge-base path, plugin name, workflow name, validation label, or file name unless the same phrase is also part of the user-approved presentation content.

For completed slides, include exact slide text and the visual direction. Chinese or dense factual text is allowed only because this output is a reference composition, not the final editable source.

If the user says source data/results must not be changed, do not ask GPT-image-2 to invent or redraw those values. Put the original charts or figures into `source_layers`, keep visible text in `text_items`, and preserve the source registry entry that explains which layers are grep-required.

Before creating `source_layers`, classify each extracted image:

- `result/data figure`: keep as source-locked image when exact values, axes, or plotted relationships must be preserved.
- `diagram/schematic/icon`: do not keep as a large source panel unless the visual itself is the evidence; extract its text into editable slide text and recreate or simplify the visual with imagegen/background design.
- `decorative or text-heavy screenshot`: do not reserve a panel; summarize or delete.

For figure-first slides, make one primary visual area per message where possible. If multiple result figures are needed, split the story across slides rather than shrinking every figure until none is readable.

For background slides, use the completed slide as the edit input and require:

```text
Remove all readable text, letters, numbers, labels, captions, and watermarks only.
Preserve every non-text visual element exactly: layout, illustrations, charts without labels, icons, shapes, lighting, texture, color, perspective, and spacing.
The output must be a text-free background for editable PowerPoint overlays.
The only difference from the input should be the absence of text.
```

Do not create a background from a new generation prompt if a completed image exists; use edit so the image geometry stays aligned.

Never populate `completed/` from PPTX page screenshots, PDF screenshots, LibreOffice renders, PowerPoint exports, local PIL drawing, or deterministic compositor output. `completed/` is the GPT-image-2 visual target; rendered PPTX/PDF pages are only QA outputs under `reports/` or accepted manual exports under `pptx/`.

If any stage violates this order, automatically discard every downstream artifact from the first invalid stage and restart from the last valid checkpoint. Examples: if `completed/` is a PPTX screenshot, delete `completed/`, `background/`, `analysis/`, `pptx/`, and QA reports; if `background/` is a local template or repeated batch, delete `background/`, `analysis/`, `pptx/`, and QA reports; if source locking changes provenance, re-register both completed and background manifests before continuing.

## Imagegen CLI

Default path:

```bash
native Codex image_gen tool
```

Create queues:

```bash
python skills/image2slides/scripts/image2slides.py queue --project <deck-dir>
```

Dry-run API payloads:

```bash
python skills/image2slides/scripts/image2slides.py imagegen --project <deck-dir> --phase completed --dry-run
python skills/image2slides/scripts/image2slides.py imagegen --project <deck-dir> --phase background --dry-run
```

The API CLI fallback is optional and only for explicit API/SDK runs:

```bash
python skills/image2slides/scripts/image2slides.py imagegen --project <deck-dir> --phase completed --execute
python skills/image2slides/scripts/image2slides.py imagegen --project <deck-dir> --phase background --execute
```

When using Codex native `image_gen` instead of the fallback CLI, copy the native GPT-image-2 completed outputs into `completed/`, edit those completed images with native `image_gen` into text-free backgrounds, then register both batches before analysis:

```bash
python skills/image2slides/scripts/image2slides.py register-completed --project <deck-dir>
# native image_gen edit: completed/slide_XX_completed.png -> background/slide_XX_background.png
python skills/image2slides/scripts/image2slides.py register-background --project <deck-dir>
python skills/image2slides/scripts/image2slides.py compose-source-locked --project <deck-dir>
```

## Keynote Lessons Carried Forward

The `szhct-youqin-keynote` project showed the right deck boundary:

- keep important Chinese/factual text editable in PowerPoint
- use imagegen for visual direction, backgrounds, components, and disposable composition sketches
- keep source policy strict: no confirmed fact enters the final deck without a source registry entry or user confirmation
- use native PowerPoint text overlays and run render/QC before completion

Image2Slides intentionally differs by allowing full-slide completed references, then converting those references into editable PPTX via matched text-free backgrounds and pixel-diff text placement.
